Claude Code: Tools and Terms

A dictionary-style reference for beginners. Each entry defines one concept, explains it in plain language, and shows what it looks like in practice. Open an entry or browse by section.

Core Concepts

TermOne-line gist
Claude CodeClaude Code is an agentic coding environment — Claude running in your terminal, able to read your files, run commands, and make changes to your project while you watch or step away.
Context WindowThe context window is the fixed amount of text Claude can hold in view at one time — it fills up, and when it does, Claude's performance degrades.
CLAUDE.mdCLAUDE.md is a special markdown file that Claude reads at the start of every conversation — a place to write down standing instructions so you don't have to re-explain them every session.
CLAUDE.local.mdCLAUDE.local.md is a personal, private version of CLAUDE.md for one project — for notes you don't want to share with your team via git.
Auto MemoryAuto memory is a feature where Claude automatically saves notes about your project and preferences across sessions — without you writing anything.
Skills (.claude/skills/)Skills are reusable workflows and domain knowledge you give Claude — stored in .claude/skills/, each skill is a folder with a SKILL.md file that Claude applies automatically when relevant.
Agents / Subagents (.claude/agents/)Subagents are specialized versions of Claude that run in their own context window — Claude can delegate tasks to them so the research or analysis doesn't clutter your main conversation.
Rules (.claude/rules/)Rules are instruction files in .claude/rules/ — a way to organize project instructions into separate topic files instead of one long CLAUDE.md.
Path-Specific RulesPath-specific rules are rule files with a paths: field in their frontmatter — they only load into context when Claude is working with files matching the specified pattern.
Plan ModePlan mode lets Claude read files and think through a problem without making any changes — you review and approve before anything gets written.
PermissionsPermissions control what Claude Code is allowed to do — write files, run commands, use tools — and how much it asks you before doing it.
HooksHooks are shell commands that run automatically at specific points in Claude's workflow — they're deterministic, unlike CLAUDE.md instructions which Claude may or may not follow.
MCP ServersMCP (Model Context Protocol) servers are external tools Claude can connect to — things like Notion, Figma, your database, or a GitHub integration.
PluginsPlugins bundle skills, hooks, subagents, and MCP servers into a single installable unit — browse and install them from the marketplace with /plugin.
Non-Interactive ModeNon-interactive mode lets you run Claude from the command line without starting a session — useful for scripts, CI pipelines, and automated workflows.
CheckpointsEvery prompt you send creates a checkpoint — a snapshot of your conversation and files that you can restore at any time.
SessionsClaude Code saves conversations locally so you can pick up exactly where you left off — resume the most recent session with claude --continue, or choose from a list with claude --resume.

Slash Commands

TermOne-line gist
/init/init generates a starter CLAUDE.md file for your current project by analyzing your codebase.
/clear/clear resets the context window entirely — use it between unrelated tasks to start fresh.
/compact/compact summarizes the current conversation to free context space while keeping the session going — unlike /clear, you don't lose the thread.
/memory/memory lists all CLAUDE.md, CLAUDE.local.md, and rules files loaded in your current session, and provides a link to open the auto memory folder.
/permissions (command)/permissions opens the permission configuration UI — use it to allowlist specific commands so Claude stops asking for approval every time.
/rewind/rewind opens a checkpoint menu where you can restore your conversation, your files, or both to any earlier state.
/rename/rename lets you give the current session a descriptive name so you can find it later.
/btw/btw lets you ask a quick side question that appears in a dismissible overlay and never enters conversation history.
/sandbox/sandbox enables OS-level isolation — Claude can work more freely within defined boundaries without the risk of affecting the rest of your system.

Prompting Patterns

TermOne-line gist
Give Claude a way to verify its workThe single highest-leverage thing you can do.
Explore first, then plan, then codeSeparate research and planning from implementation to avoid solving the wrong problem.
Provide specific contextThe more precise your instructions, the fewer corrections you'll need.
Reference files with @Use @ to pull files into context directly — Claude reads the file before responding.
Course-correct earlyThe best results come from tight feedback loops.
Use subagents for investigationDelegate research to subagents so the exploration doesn't consume your main context.
